1. Quy Trình Điều Khiển Thiết Bị Cuối - Cuối (End-to-End)

Hướng dẫn: Cuộn chuột để Zoom, nhấn giữ để kéo sơ đồ để xem chi tiết các bước.

sequenceDiagram
    autonumber
    actor User as 👤 Người dùng
    participant APP as 📱 Flutter App
    participant Flask as 🐍 Flask API (JWT)
    participant DB as 🗄️ PostgreSQL
    participant MQTT as 📡 MQTT Broker
    participant ESP as 🔌 ESP32 Hardware

    Note over User,ESP: Khởi tạo lệnh điều khiển thiết bị

    User->>APP: Nhấn nút "Bật Đèn"
    APP->>APP: Optimistic UI Update (Icon sáng ngay)
    
    APP->>Flask: POST /api/devices/control {device_code, action:ON}
    Note right of Flask: JWT check identity → current_user_id

    Flask->>DB: SELECT device WHERE code='den' AND user_id=current_user
    DB-->>Flask: Hợp lệ (Device belongs to user) ✅
    
    Flask->>DB: Ghi log thao tác (device_logs)
    Flask->>MQTT: Publish topic 'home/control/den' {action:ON}
    
    MQTT->>ESP: Chuyển tiếp lệnh JSON
    Flask-->>APP: Phản hồi 200 OK (Đã gửi lệnh)

    Note over ESP: Xử lý tại tầng phần cứng
    ESP->>ESP: digitalWrite(GPIO17, HIGH) → Relay ON
    ESP->>MQTT: Feedback trạng thái thực 'home/status/den' {status:ON}

    MQTT->>Flask: Nhận phản hồi trạng thái
    Flask->>DB: Cập nhật status thực tế vào DB
    Flask->>APP: [Socket.IO] emit 'device_update' (Room theo user_id)

    APP->>APP: Đồng bộ UI với trạng thái thực tế ✅
2. Kiến Trúc Cô Lập Dữ Liệu Đa Người Dùng (Multi-tenancy)

Sơ đồ này giải thích cách hệ thống ngăn chặn User A điều khiển thiết bị của User B.

flowchart TB
    subgraph Users ["Nhóm Người Dùng Hệ Thống"]
        direction LR
        subgraph UserA ["👤 Gia đình Nguyễn (ID: 7)"]
            UA_T["🔑 Token User_7"]
            UA_D["🏠 Thiết bị Nhà A"]
            UA_S["🔌 Socket Room '7'"]
        end

        subgraph UserB ["👤 Gia đình Trần (ID: 15)"]
            UB_T["🔑 Token User_15"]
            UB_D["❄️ Thiết bị Nhà B"]
            UB_S["🔌 Socket Room '15'"]
        end
    end

    subgraph Backend ["🛡️ Flask Security & Isolation Layer"]
        direction TB
        AUTH{"@jwt_required
Identity Check"}
        FILTER["SQL Logic:
WHERE user_id = current_identity"]
        DENY["❌ 403 Forbidden
(Truy cập trái phép)"]
    end

    subgraph Data ["🗄️ Database Tầng Chứa Dữ Liệu"]
        T1[(Bảng Devices - user_id FK)]
        T2[(Bảng Health - user_id FK)]
    end

    %% Kết nối logic
    UA_T & UB_T ==> AUTH
    AUTH --> FILTER
    
    FILTER -->|"user_id=7"| T1 & T2
    FILTER -->|"user_id=15"| T1 & T2

    %% Minh họa ngăn chặn
    UA_T -.->|"Cố ý gửi lệnh điều khiển Device B"| FILTER
    FILTER -.->|"Truy vấn trả về NULL"| DENY

    %% Phân phối thông báo Real-time
    FILTER -- "Chỉ gửi đến" --> UA_S
    FILTER -- "Chỉ gửi đến" --> UB_S
