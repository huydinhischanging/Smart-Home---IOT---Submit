Forgot Password & Security Recovery Flow
Xác thực Token • Hashing SHA-256 • Bcrypt Updates
SECURE PROTOCOL
sequenceDiagram
    autonumber
    actor User as 👤 Người dùng
    participant FE as 📱 Frontend (JS/Flutter)
    participant BE as 🐍 Backend (Flask API)
    participant DB as 🗄️ PostgreSQL
    participant SMTP as 📧 Gmail SMTP

    Note over User,FE: Bước 1: Yêu cầu khôi phục
    User->>FE: Nhấn "Quên mật khẩu" & nhập Email
    FE->>BE: POST /api/auth/forgot-password {email}
    
    BE->>DB: Tìm User theo Email
    
    alt Không tìm thấy hoặc Inactive
        DB-->>BE: null / is_active=false
        BE-->>FE: 404/403 Error Message
        FE-->>User: Hiển thị thông báo lỗi
    else Email hợp lệ
        DB-->>BE: User Object
        BE->>BE: Tạo Secure Token (secrets.token_urlsafe)
        BE->>BE: Hash Token (SHA-256)
        BE->>DB: Lưu Hash & Hạn dùng (+30 phút)
        
        BE->>SMTP: Gửi Email chứa Token khôi phục
        
        alt SMTP Gửi thành công
            SMTP-->>BE: OK
            BE-->>FE: 200 "Email Sent"
            FE-->>User: "Vui lòng kiểm tra hộp thư"
        else Lỗi SMTP (Fallback Dev Mode)
            BE-->>FE: 200 {reset_token: "..."}
            FE-->>User: Hiển thị mã trực tiếp trên màn hình
        end
    end

    Note over User,FE: Bước 2: Đặt lại mật khẩu mới
    User->>FE: Nhập Token từ Email + Mật khẩu mới
    FE->>BE: POST /api/auth/reset-password {token, new_pass}
    
    BE->>BE: Hash(Token) & So khớp
    BE->>DB: Truy vấn Token Hash & Kiểm tra expires_at
    
    alt Token sai hoặc Hết hạn
        DB-->>BE: Invalid / Expired
        BE-->>FE: 401 Unauthorized
        FE-->>User: "Mã không hợp lệ hoặc đã hết hạn"
    else Xác thực thành công
        BE->>DB: Cập nhật User Password (Bcrypt)
        BE->>DB: Đánh dấu Token đã sử dụng (is_used=true)
        BE->>BE: Tạo JWT Token mới
        BE-->>FE: 200 {jwt_token, message: "Success"}
        FE-->>User: ✅ Thành công! Đang chuyển hướng...
    end
