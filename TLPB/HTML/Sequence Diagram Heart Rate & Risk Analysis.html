Sequence Diagram: Heart Rate & Risk Analysis

sequenceDiagram
    autonumber
    participant H6 as [Device] Coospo H6
    participant BLE as [Bridge] Python Bleak
    participant MQTT as [Broker] MQTT
    participant BE as [Server] Backend
    participant HRV as [AI] HRV/Anomaly Engine
    participant UI as [Client] Web/Mobile

    note over H6, BLE: Giao tiếp qua giao thức Bluetooth Low Energy
    H6->>BLE: BLE packet (0x2A37 Heart Rate Measurement)
    activate BLE
    BLE->>BLE: Decode Byte Stream to BPM
    BLE->>MQTT: Publish home/sensors/heart_rate
    deactivate BLE

    MQTT->>BE: Deliver Message (BPM, Timestamp)
    activate BE
    BE->>BE: Persist HR Record (PostgreSQL)
    
    note right of BE: Trigger HRV Pipeline
    BE->>HRV: Update Sliding Window Samples
    activate HRV
    HRV->>HRV: Compute RR Intervals (60k/BPM)
    HRV->>HRV: Analysis (RMSSD, SDNN, pNN50)
    HRV-->>BE: Risk Assessment Result
    deactivate HRV

    BE->>UI: Socket.IO Emit (hr_alert / sensor_update)
    activate UI
    UI->>UI: Update Dashboard / Trigger Notification
    deactivate UI
    deactivate BE
